데이터베이스 관리 시스템의 등장 배경

데이터베이스가 등장하기 전, 데이터들은 파일시스템에 의해 관리 되었다. 파일시스템은 데이터를 파일로 관리하기 위해 파일을 생성, 삭제, 수정, 검색하는 기능을 제공해 주는 소프트웨어이다. 하지만 파일시스템을 이용한 데이터관리는 많은 문제점을 가지고 있었다.

파일시스템의 문제점

  • 같은 내용의 데이터가 여러 파일에 중복 저장
  • 응용 프로그램이 데이터 파일에 종속적이다.(사용하는 파일의 구조를 변경하면 응용 프로그램도 함께 변경 해야함)
  • 데이터 파일에 대한 동시 공유, 보안, 회복 기능이 부족하다.
  • 응용 프로그램 개발이 쉽지 않다.

예를들어, 고객의 연락처가 변경된 경우라던지, 고객아이디를 영어와 숫자를 섞어 5자 이상 제약을 둔다던지 하는 상황이라면 파일시스템은 적절하지 않다.

파일시스템을 이용한 데이터관리

학생 데이터를 파일시스템으로 관리한다고 생각해보자. 졸업여부를 관리하는 파일이 있고, 성적을 관리하는 파일이 있다고 가정한다. 여러개의 파일중에 중복되는 데이터들이 있을 것이다.

엑셀 파일안에 데이터를 입력한 후 프로그램에서 데이터를 읽어오기 위해 csv형태로 저장했다.

이제 데이터를 읽고, 수정할 때 어떠한 어려움이 있을까? 어떤 학생의 졸업여부와 성적을 알고 싶을 때 각각의 파일에서 데이터를 읽어와야 한다. 

<psuedo code>

buffer1 = read(file1.csv);
buffer2 = read(file2.csv);
while (!feof(buffer1)){
	line = fgets(buffer1, 1024)
	if(찾는 학생의 학번 == line[0])
		print(찾는학생의 졸업여부)
}

while (!feof(buffer2)){
	line = fgets(buffer2, 1024)
	if(찾는 학생의 학번 == line[0])
		print(찾는학생의 성적)
}

이런 식이다. (프로그래밍 언어로 제대로 작성하면 더욱 복잡할 것이다.) 뿐만 아니라 중복된 데이터를 수정할 때는 일이 더 커진다. 하나의 파일에서 수정을 하면 다른 파일에서도 똑같은 내용으로 수정해야하는데, 여간 쉬운일이 아닐것이다.

결론적으로, 파일시스템은 데이터의 중복성, 일관성과 무결성을 유지하기 힘들다.

데이터베이스 관리 시스템(DBMS)등장

이러한 파일시스템의 다양한 문제로 인해 데이터베이스 관리 시스템(DBMS)이 등장했다. DBMS는 데이터베이스를 관리, 운영하는 소프트웨어이다. DBMS을 이용함으로서 응용프로그램을 개발할 때 더욱 쉽게 데이터를 관리할 수 있게 되었고, 보안능력도 향상되었다. 대표적인 DBMS로는 MySQL, MariaDB, MongoDB등이 있다.

데이터베이스 관리 시스템의 주요기능

정의 기능: 데이터베이스 구조를 정의하거나 수정할 수 있다.

조작 기능: 데이터를 삽입,삭제,검색하는 연산을 할 수 있다.

제어 기능: 데이터를 항상 정확하고 안전하게 유지할 수 있다.
(일관성 및 무결성 유지, 장애회복, 데이터접근허용, 보안유지, 동시접속처리).

데이터베이스 관리 시스템의 발전 과정

1세대: 네트워크 DBMS, 계층 DBMS

네트워크 DBMS : 데이터베이스를 그래프 형태로 구성한 방식이다. 구조가 복잡하고, 변경하기 어렵다는 단점이 있다.

계층 DBMS : 데이터베이스를 트리 형태로 구성한 방식이다. 이것 또한 표현하기 어렵고, 구조변경이 어렵다는 단점이 있다.

2세대: 관계 DBMS

관계 DBMS : 데이터베이스를 테이블 형태로 구성한 방식이다. 직관적이고, 유지보수하기 쉽다. 대용량의 데이터를 체계적으로 관리할 수 있고 데이터의 무결성도 잘 보장된다. 하지만 시스템 자원을 많이 차지하여 시스템이 전반적으로 느려지는 단점이 있다.

3세대: 객체지향 DBMS, 객체관계 DBMS

객체지향 DBMS : 객체를 이용해 데이터베이스를 구성한 방식이다.객체지향DBMS와 관계형DBMS의 가장 큰 차이는 DB의 저장에 있다. 관계형DB의 경우 미리 정의된 데이터 형식에 맞춰 데이터만을 저장할 수 있는 데 비해 객체지향DBMS의 경우 ‘데이터와 연관된 프로그램’인 ‘메소드’를 함께 저장해 사용할 수 있다는 점이 가장 큰 차이점이다. 하지만 검색 성능이 느리고, 대규모 트랜잭션 처리 성능이 떨어진다는 단점이 있다.

객체관계 DBMS : 객체지향DBMS과 관계DBMS를 합친 것이다. 각각의 장점들을 가져옴으로서 성능을 강화했다.

4세대: NoSQL, NewSQL

NoSQL DBMS : Not Only SQL의 약자로서, 관계형 데이터베이스와 반대되는 개념이다. 데이터와 테이블 간의 관계를 정의하지 않고 정해진 스키마가 없어 자유롭게 데이터를 저장할 수 있다. 데이터 구조를 미리 정해두지 않아서 유연하고, 확장성이 뛰어나다는 장점이있다. SNS의 발전으로 인한 비정형 데이터의 증가로 NoSQL이 환영받고 있다.

NewSQL DBMS : 관계 DBMS의 장점과 NoSQL의 장점을 모아 탄생했다.

SQL이란? 데이터베이스를 조작하는 언어로 DBMS 제작 회사와 독립적이다. 대부분의 DBMS 제작회사가 표준 SQL을 준수하려고 노력하지만 각 DBMS마다 특징이 있기 때문에 완전히 통일되기는 어렵다.
