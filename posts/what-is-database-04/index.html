관계 데이터 모델 개념

관계 데이터 모델은 1970년대 등장 했지만, 여러 장점들 때문에 지금까지 널리 사용되고 있다. 매우 직관적이어서 데이터베이스에 대해 모르는 사람도 쉽게 해석할 수 있고, 구조도 단순해서 조작하기 편리하다. 관계 데이터 모델을 이용해 데이터를 관리하기 위해선 용어를 알아야한다.

릴레이션(relation)

하나의 개체에 관한 데이터를 2차원 테이블의 구조로 저장한 것이다. 파일 관리 시스템 관점에서 파일(file)에 대응한다. 데이터베이스 안에서 여러 릴레이션들이 존재할 수 있다. 예를 들어, 쇼핑몰에서는 고객 릴레이션, 상품 릴레이션, 주문 릴레이션 등이 있을 수 있다.

속성(attribute)

릴레이션의 열을 나타낸다. 파일 관리 시스템 관점에서 필드(field)에 대응한다. 하나의 릴레이션에 같은 속성이 여러 개 올 수 없다. 왜냐하면 데이터베이스의 데이터들은 중복되지 않는 특성을 가지고 있기 때문이다. 또한 같은 속성이 여러 개라면, 사용자가 어떤 데이터를 원하는지 알 방법이 없다. 그러므로 속성의 이름은 중복될 수 없다. 또한 같은 속성 안에 여러 개의 값이 올 수 없다. 이 부분은 데이터 모델의 제약에서 자세히 다루겠다.

튜플(tuple)

릴레이션의 행을 나타낸다. 파일 관리 시스템 관점에서 레코드(record)에 대응한다. 여러 속성들을 담고 있는 데이터로, 추가, 수정, 삭제될 수 있는 동적 데이터이다. 고객 관리 릴레이션에서 한 명의 회원 데이터를 나타낸다.

도메인(domain)

쉽게 말하면, 속성의 데이터타입을 말한다. 예를 들어, 상품의 가격 속성은 데이터타입을 int로 하는 것이 좋을 것이다. 소수점을 입력했을 때 예외처리를 할 수 있고(데이터 무결성), character로 도메인을 설정했을 때 가격 연산이 힘들어 질 수 있기 때문이다. 그러므로 도메인을 어떻게 설정해야 하는지도 깊게 고려해야한다.

차수(degree)

하나의 릴레이션안에 속성의 개수를 의미한다.

카디널리티(cardinality)

하나의 릴레이션안에 튜플의 개수를 의미한다.

릴레이션 스키마

릴레이션의 속성들의 집합이다. 테이블에서 첫번째 행을 의미하기도 하며, 한번 정의하면 수정하지 않는 이상 정적으로 남아있다.

릴레이션 인스턴스

릴레이션의 어느 한 시점에 존재하는 튜플들의 집합이다. 사용자의 데이터가 추가, 삭제, 수정 되기 때문에 동적인 특징이 있다.

지금까지 하나의 릴레이션 안에서 사용되는 용어들을 알아봤다. 하나의 릴레이션이 아니라 더 넓은, 데이터베이스 관점에서 봤을 때 사용되는 용어들을 알아보자.

데이터베이스 스키마
데이터베이스의 전체 스키마다. 쇼핑몰을 예로 들면, 고객 스키마, 상품 스키마, 주문 스키마 등의 집합이 데이터베이스 스키마라고 볼 수 있다.

데이터베이스 인스턴스
전체 데이터베이스에서 인스턴스들의 집합이다.

용어를 알아봤으니 관계 데이터 모델의 특징을 살펴보자.

릴레이션 특징

하나의 릴레이션 안에는 동일한 튜플이 두 개 이상 올 수 없다. 데이터의 중복을 피하기 위해 DBMS를 사용하는데, 똑같은 튜플이 있다는 것은 데이터베이스 사용목적에 어긋나기 때문이다.

두 번째 특징은, 튜플과 속성은 순서 상관이 없다는 것이다. 고객관리 릴레이션에서 이름이 먼저 나오나, 전화번호가 먼저 나오나 데이터를 관리하는데 전혀 이상이 없다. 물론 미관상 이름이 먼저 나오는 것이 바람직하지만, 그것은 단지 미관상일 뿐이다. 튜플도 마찬가지다. 튜플 사이엔 우선순위가 필요 없기 때문에 순서는 무의미하다.

세 번째 특징은, 도메인에 형식을 맞춰 속성 값을 입력해야한다는 것이다. 이것은 데이터무결성과 관련된 것으로, 전화번호 입력 칸에 글자를 넣는다 던가, 이름에 숫자를 넣는 것을 방지해준다. 또한 속성 값에는 무조건 단일 값을 넣어주어야 한다. 물론 속성 값으로 배열을 넣을 수 있기는 한데, 결국 배열도 원소 하나로 간주한다면, 단일 값이라 볼 수 있을 것이다. 속성에 값을 여러 개 넣는 다면 어떤 데이터를 가져와야 하는지 알 수 없을 뿐만 아니라, 어떤 속성엔 값이 하나인데, 어떤 속성엔 값이 여러 개가 들어갈 수 있으므로 데이터 일관성도 만족하지 못한다.

키(KEY)

관계 데이터 모델에서 ‘키(key)’는 중요한 역할을 한다. 특히 값을 참조하기 위해선 ‘키’가 꼭 필요하다. 키는 릴레이션에서 튜플들을 유일하게 구별하는 속성 또는 속성들의 집합이다. 왜 ‘키’가 필요할까? 회원정보를 찾으려고 하는데 이름으로 찾게 되면 중복된 이름이 있을 수 있다. 그러면 내가 찾고 싶은 회원이 누구인지 알 수 없다. 그렇기 때문에 데이터를 구별하기 위해선 ‘키’가 반드시 필요하다. 데이터를 찾아줄 열쇠라고 보면 된다. 키는 여러 종류가 있는데 하나하나 알아보도록 하자.

슈퍼키

유일성을 만족하는 속성 또는 속성들의 집합이다. 예를 들어, 슈퍼키는 주민번호, 회원아이디가 될 수 있고, 대학교에서는 학번이 슈퍼키가 될 수 있다. 대한민국에 같은 주민번호를 가지고 있는 사람은 없을 것이다. 슈퍼키는 ‘속성들의 집합’이기 때문에 주민번호에 아이디를 합한다 던지, 주민번호에 이름을 합한다 해도 슈퍼 키가 될 수 있다. 그런 모든 집합은 다 슈퍼키이다. 데이터들을 구별 할 수 있는 것들은 모두 슈퍼키라고 볼 수 있다.

후보키

그렇다면 슈퍼키는 무수히 많아질 것이다. 그 슈퍼키들 중에 최소성을 만족하는 키가 후보키들이 된다. 주민번호와 이름을 합한 키는 후보키가 될 수 없다. 최소성을 만족하지 못했기 때문이다. 그러므로 유일성과 최소성을 만족하는 주민등록번호는 후보키가 될 수 있다. 물론 주소+이름도 후보키가 될 수 있다. 유일성을 만족하는 가장 최소한의 키의 조합이기 때문이다. 이런 키를 ‘복합키’ 라고 부른다.

기본키

여러 후보키들 중에서 기본적으로 사용할 키를 기본키라고 부른다. 기본키는 무조건 한 개 여야 한다. Null값은 당연히 될 수 없고, 후보키가 하나뿐이라면 그 후보키를 기본키로 사용하면 된다. 추후 문제가 발생할 일이 없는 키를 기본키로 설정하는 것이 좋다. 또한 기본키는 수정 되어선 안 된다.

대체키

대체키는 기본키로 선택 받지 못한 후보키들을 말한다.

외래키

다른 릴레이션의 기본키를 참조하는 속성 또는 속성들의 집합이다. 데이터베이스 안에는 여러 릴레이션이 있을 수 있다. 한 릴레이션에서 다른 릴레이션의 값을 참조하고 싶을 때 외래키를 사용한다. 예를 들어 ‘회원 릴레이션’과 ‘게시 글 릴레이션’이 있을 때 회원아이디라는 외래키를 이용해 회원이 쓴 게시 글들을 참조할 수 있다. 이 때 회원 아이디가 외래키가 된다. 외래키를 이용해 릴레이션의 관계를 표현할 수 있다. 이때 각 릴레이션 마다 속성 이름은 달라 질 수 있다.

관계 데이터 모델 제약

무결성 제약조건이란 데이터의 무결성을 보장하고 일관된 상태를 유지하기 위한 규칙이다. 데이트베이스가 삽입, 삭제, 수정, 연산으로 상태가 변하더라도 무결성 제약조건은 반드시 지켜져야 한다. 무결성을 유지하기 위한 제약조건들은 여러 개가 있다.

개체 무결성 제약조건

기본키를 구성하는 모든 속성값은 null값이나 중복되는 값을 가질 수 없다.

참조 무결성 제약조건

외래키는 참조할 수 없는 값을 가질 수 없다. 외래키 속성이 널 값을 가진다고 해서 참조 무결성 제약조건을 위반한 것은 아니다. 즉 참조 무결성 제약조건을 만족하려면 외래키의 값이 참조하는 릴레이션의 어떤 튜플의 기본 키 값과 같거나 참조하지 않을 경우 null값을 가지면 된다.

도메인 무결성 제약조건

속성의 도메인에 벗어나는 속성 값을 가질 수 없다.

고유 무결성 제약조건

특정 속성에 대해 고유한 값을 가지도록 조건이 주어진 경우, 그 속성값은 모두 달라야 한다.

Null 무결성 제약조건

특정 속성값에 NULL 이 올 수 없다는 조건이 주어진 경우, 그 속성값은 NULL 값이 올 수 없다는 제약조건을 말한다.

키 무결성 제약조건

한 릴레이션(테이블)에는 최소한 하나의 키가 존재해야 한다.

이 조건들만 올바르게 충족한다면 데이터베이스의 무결성은 보장된다.

관계 데이터 모델 연산

지금까지 관계 데이터 모델 제약조건에 대해 알아보았다. 지금부터는 관계 데이터 모델 연산에 대해 알아보도록 하겠다.

관계 데이터 연산은 크게 일반 집합 연산자와 순수 관계 연산자로 나뉜다. 일반 집합 연산자는 단어 그대로 집합 개념이 쓰인다. 합집합, 교집합 및 차집합은 두 릴레이션의 속성이 같은 상태에서 집합 연산을 수행 후 새로운 릴레이션을 반환한다. 반면 카티션 프로덕트(cartesian product)는 조금 생소할 것이다. 집합의 곱셈과 같은 개념인데, 두 개의 릴레이션의 튜플들을 모두 연결하서 만들어진 릴레이션을 반환한다.

일반집합연산자는 짧게 알아봤고, 순수 관계 연산자에 대해 알아보자

셀렉션(Selection)

릴레이션에서 조건을 만족하는 튜플을 반환하는 연산자이다. 하나의 릴레이션을 대상으로 하고, σ조건식(릴레이션) 으로 표현한다. 데이터 언어로 (릴레이션) where (조건식) 으로 표현할 수 도 있다.

예를 들어, “고객 릴레이션에서 회원등급이 ‘골드’인 튜플을 검색하시오” 라고 한다면 σ 회원등급=골드(고객) 또는 고객 where 회원등급=골드 라고 하면 된다. 교환법칙이 성립한다.

프로젝트(Project)

선택한 속성으로만 만들어진 릴레이션을 반환하는 연산이다. 수학적 표기법으로 𝜋속성리스트(릴레이션), 데이터 언어적으로 릴레이션[속성리스트]라고 하면 된다.

조인(Join)

두 릴레이션의 공통 속성을 기준으로 속성 값이 같은 투플을 수평으로 결합하는 연산이다. 조인을 수행하기 위해서는 두 릴레이션의 조인에 참여하는 속성이 서로 동일한 도메인으로 구성되어야 한다. 조인 연산의 결과는 공통 속성의 속성 값이 동일한 투플만을 반환한다. 하나의 릴레이션으로 원하는 데이터를 얻을 수 없어 두 개의 릴레이션이 필요할 때 사용된다.

세타조인은 조건에 맞는 튜플만 반환한다. =, ≠, ≤, ≥, ＜, ＞ 기호를 사용하여 조건식을 만든다.

동등조인은 세타조인에서 = 기호만 사용한 조인을 말한다. 보통 조인연산이라 하면 동등조인을 지칭한다.

자연조인은 위 사진과 같이, 중복된 속성을 제거한 릴레이션을 반환한다.

외부조인은 자연조인 시, 실패한 튜플들을 보여주되 값이 없는 대응속성에는 NULL값을 채워서 반환한다. 모든 속성을 보여주는 기준 릴레이션의 위치에 따라, 왼쪽 외부조인, 오른쪽 외부조인, 완전 외부조인으로 나뉜다.

세미조인은 자연조인을 한 후, 한 쪽 릴레이션의 결과만 반환한다. 기호에서 닫힌 쪽만 반환한다. 조건을 만족한 튜플 중에 굳이 다른 정보들을 보고 싶지 않을 때 사용한다.

디비전(Devision)

(릴레이션1)÷(릴레이션2)으로 표기하고 각 릴레이션에 중복된 속성을 기준으로 (사진에서는 B속성) 릴레이션 2의 속성 값이 있는 릴레이션1의 속성들을 반환하면 된다.
